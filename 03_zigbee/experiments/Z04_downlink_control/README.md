# Z4 Zigbee下行控制与智能开关实验

## 基本信息

| 项目 | 内容 |
| --- | --- |
| 方向 | Zigbee |
| 类型 | 进阶 |
| 难度 | 中 |
| 主要设备 | CC2530节点、PC/网关、LED/继电器或模拟执行器 |
| 协议/接口 | MQTT→串口→Zigbee |
| 负责人 | 王道嘉 |
| 协作人 | 方宏波 |
| 成果物 | 代码+指导书+PPT |

## 实验目标

从MQTT命令控制Zigbee终端LED/继电器，形成命令格式与回执机制

## 备注

与Z6共享下行命令模型

## 目录结构

```text
Z04_downlink_control/
  src/      实验源码
  config/   配置文件
  docs/     实验指导书、说明文档
  slides/   PPT 或汇报材料
  tests/    测试脚本、测试记录
  assets/   接线图、截图、素材
```

## 每周任务

| 周次 | 工作重点 | 主要任务 | 本周交付物 | 验收标准 |
| --- | --- | --- | --- | --- |
| 第1周 | Zigbee传感与控制 | 熟悉CC2530节点与上位机串口数据格式；设计传感上报、下行控制、智能家居联动的topic和payload草案。 | Z3/Z4/Z7方案表、MQTT topic草案、传感/控制数据字段说明。 | 完成一条模拟传感数据从串口到PC脚本的解析demo。 |
| 第2周 | 实现传感上报与下行控制基础 | 实现Zigbee传感/模拟数据上报到PC脚本；实现MQTT命令到Zigbee终端LED/执行器控制的基础链路。 | Z3/Z4代码、payload样例、指导书初稿、PPT提纲。 | MQTT中可看到Z3上行消息；发送Z4命令后终端状态改变并返回回执。 |
| 第3周 | 完成Zigbee智能家居综合场景 | 把Z3上行和Z4下行合并为Z7场景：环境数据触发告警，MQTT/VSOA命令控制终端。 | Z7综合代码、场景指导书初稿、PPT主体、演示脚本。 | 能演示从传感数据到告警/控制的闭环。 |
| 第4周 | Zigbee传感控制与家居场景定稿 | 完成Z3/Z4/Z7最终材料；统一topic和payload；准备5分钟课堂演示。 | Z3/Z4/Z7最终代码、指导书定稿、PPT定稿、演示脚本。 | Z7综合演示稳定；PPT可直接用于课堂讲解。 |
